# Opdracht
1. Lees [Namespace in C++](https://www.geeksforgeeks.org/cpp/namespace-in-c/)  in ieder geval deel 1. Er wordt vooral de nadruk gelegd op het voorkomen van conflicten in namen. Dat is natuurlijk ook helemaal waar, maar namespaces (of packages als meer algemene term), kunnen je ook helpen overzicht te houden over je code als je project erg groot wordt.
Dus, namespaces kunnen helpen je code te organiseren. Lees ook [Package Diagram | Introduction, Elements, Use Cases and Benefits](https://www.geeksforgeeks.org/system-design/package-diagram-introduction-elements-use-cases-and-benefits/), het verschil tussen de vormen van relaties tussen packages is nu niet belangrijk.

Bekijk je ontwerp van je project (wat naast de lessen loopt) tot nu toe. Bedenk hoe je je ontwerp verder kan structureren door het in te delen in packages.

2. Vingeroefening met design patters: Implementeer een Singleton, en schrijf code om te testen dat het goed werkt. Liefst zie ik hier een relatie met je ontwerp van het project. Dus is er een klasse die je als singleton zou kunnen implementeren? Of waarom niet?
